# V1.1 Architecture: Epipolar Cross-Attention + Fourier PE

## Motivation

V1 uses Inverse Splat (LSS-based lift-splat) to transform nuScenes 5-camera features
into Physical AI 3-camera feature maps. Key limitations:
- Depth discretization (64 bins) causes information loss
- `scatter_add_` quantization artifacts and per-batch GPU loops
- Depth prediction quality is a training bottleneck
- Large intermediate tensor: B*N*D*C*fH*fW ~ 829M floats

## V1.1 Changes

1. **Replace Inverse Splat** with Epipolar-Guided Cross-Attention
   - Precompute epipolar sample coordinates at init (stored as buffers)
   - Cross-attention: target queries attend to source features sampled along epipolar lines
   - 3 transformer layers, 8 heads, 256-dim

2. **Upgrade PE** from Plucker MLP to 3-component Fourier PE system
   - FourierRayPE: Fourier encoding (L=10) on 6D Plucker coords -> MLP -> 256-dim
   - Target3DPE: Same encoding for target camera ray directions (precomputed buffer)
   - CameraIDEmbedding: Learnable per-camera embeddings (5 source + 3 target)

3. **Remove Image Reconstruction** (Stage 2)
   - No ImageDecoder, no ImageLoss, no VGGPerceptualLoss

4. **Simplify Depth** to lightweight auxiliary loss
   - 1x1 Conv on feature maps -> (B, 1, 20, 36) at feature resolution
   - Replaces full DepthDecoder

5. **No Curriculum** - single-stage training, all losses from epoch 0

## Architecture Pipeline

```
Physical AI images (3 cams, 1920x1080)
    | [GeometricTransform] (reused)
NuScenes images (5 cams, 320x576)
    | [EfficientNet-B4 + FPN] (reused)
Source features (B, 5, 256, 20, 36)
    | [FourierRayPE + CameraID] (NEW)
Source features with PE (B, 5, 256, 20, 36)
    | [EpipolarCrossAttention] (NEW)
    ^ Target queries with Target3DPE + CameraID
Target feature maps: Dict[str, (B, 256, 20, 36)]
    |-> [TokenDecoder] -> visual tokens (reused, in_channels=256)
    |-> [DepthHead: 1x1 Conv] -> depth (B, 1, 20, 36) (NEW, auxiliary)
```

## Epipolar Cross-Attention Detail

For each (target_pixel, source_camera) pair:
1. Target pixel -> 3D ray via f-theta unproject
2. Sample 32 depths (log-uniform in [1, 60] meters)
3. For each depth, compute 3D point, project to source camera
4. Result: precomputed sample grid (720, 5, 32, 2) per target camera

Attention per query: 720 x 160 = 115,200 (vs full 720 x 3600 = 2.6M -> 22x smaller)

## Loss Function

```
total_loss = token_loss + depth_weight * depth_aux_loss

token_loss = 1.0 * MSE + 0.5 * Cosine + 0.1 * KL_attention
depth_aux_loss = 0.1 * L1 + 0.05 * SILog
```
